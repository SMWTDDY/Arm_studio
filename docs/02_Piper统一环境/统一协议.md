# 统一 Piper 协议

这份协议是 `Piper_test` 和 `Arm_studio` 合并后的第一层稳定边界。目标是让真实硬件、仿真环境、录制、回放，以及后续训练都围绕同一个 env/data contract 工作。

## 环境接口

所有统一 Piper 环境都应暴露以下接口：

```text
env.unwrapped.meta_keys
env.reset() -> obs, info
env.step(action_dict) -> obs, reward, terminated, truncated, info
env.unwrapped.get_safe_action() -> action_dict
```

标准观测结构：

```text
obs/state/<state_key>              float32 数组
obs/under_control/<arm_name>       bool 数组，shape 为 (1,)
obs/rgb/<camera_role>              uint8 CHW 图像
obs/depth/<camera_role>            可选 depth 图像，通常为 uint16/float CHW
```

标准动作结构：

```text
action/<prefix>arm                 6D joint 或 pose 动作
action/<prefix>gripper             1D 夹爪动作
```

单臂环境没有前缀：

```text
state/joint_pos
state/joint_vel
state/ee_pose
state/gripper_pos
action/arm
action/gripper
```

双臂环境使用 `left_` 和 `right_` 前缀：

```text
state/left_joint_pos
state/right_joint_pos
action/left_arm
action/right_arm
```

## 配置入口

统一配置文件位于：

```text
agent_infra/Piper_Env/Config/unified_real_single.yaml
agent_infra/Piper_Env/Config/unified_real_dual.yaml
agent_infra/Piper_Env/Config/unified_sim_single.yaml
agent_infra/Piper_Env/Config/unified_sim_dual.yaml
```

统一 factory：

```python
from agent_infra.Piper_Env.Env.unified_piper_env import make_unified_piper_env

env = make_unified_piper_env(
    config_path="agent_infra/Piper_Env/Config/unified_real_single.yaml",
)
```

不连接硬件的配置检查：

```bash
python3 scripts/piper/test_unified_env.py \
  -cfg agent_infra/Piper_Env/Config/unified_real_single.yaml \
  --dry-config
```

单臂仿真检查：

```bash
python3 scripts/piper/test_unified_env.py \
  -cfg agent_infra/Piper_Env/Config/unified_sim_single.yaml \
  --steps 3
```

双臂仿真协议检查：

```bash
python3 scripts/piper/test_unified_env.py \
  -cfg agent_infra/Piper_Env/Config/unified_sim_dual.yaml \
  --dry-config
```

## 当前已实现

- `backend=real, arm_mode=single` 通过 `SinglePiperEnv` 创建真实单臂环境。
- `backend=real, arm_mode=dual` 通过 `DualPiperEnv` 创建真实双臂环境。
- `backend=sim, arm_mode=single` 通过 Arm_studio ManiSkill 单臂环境创建仿真单臂，默认加载 right hand 模型 `piper_arm_right`。
- `backend=sim, arm_mode=dual` 通过两个 Arm_studio 单臂仿真场景组合出双臂协议环境；left/right 分别加载 `piper_arm_left` / `piper_arm_right`，腕部相机外参写在各自 URDF 的 `hand_camera_joint`。
- `Env/protocol.py` 提供统一 `meta_keys`、单/双臂 key 规则和 action flatten/split helper。

## 当前限制

当前双臂仿真是协议层双臂，不是同一物理场景中的双臂。也就是说，左右臂的 state/action/camera key 已经和真实双臂一致，但左右臂暂时不共享同一个 ManiSkill scene，也不会真实碰撞或共同操作同一物体。

## 后续工作

1. 新增真正共享场景的双臂 ManiSkill 环境。
2. 将录制、回放、采集脚本统一改成读取 `unified_*.yaml`。
3. 补旧 Arm_studio HDF5 到统一 raw H5 协议的转换器。
4. 对真实双臂、仿真双臂分别跑 H5 录制和 replay round-trip。
5. 训练部分后续再接入统一协议，目前还未合并。
