# 目录约定

本项目后续默认只保留两类运行产物根目录：

```text
datasets/   数据资产：采集数据、转换后的 H5/LeRobot、训练缓存、在线 replay
outputs/    运行输出：checkpoint、推理结果、debug 图、导出视频、评估可视化
```

## datasets

推荐结构：

```text
datasets/
  piper/          Piper 真实/仿真 H5 与 LeRobot 数据
  realman/        历史 Realman 数据与合并结果
  online_rl/      HITL / online runner 采样轨迹
  pipeline_tests/ pipeline smoke test 采集数据
  unified_*       标准化后的训练数据集或图像缓存
```

Piper recorder、`scripts/piper/single|dual|sim`
默认写入 `datasets/piper`。

单条轨迹文件命名统一为：

```text
<robot>_<control_mode>_<backend>_<trajectory_id>.hdf5
```

其中 `backend` 只使用 `real` 或 `sim`，`trajectory_id` 使用三位数字递增。
示例：

```text
piper_joint_sim_000.hdf5
piper_joint_real_000.hdf5
realman_joint_pos_real_000.hdf5
```

## outputs

推荐结构：

```text
outputs/
  checkpoints/       训练得到的策略、critic、config snapshot
  frames/            渲染和诊断保存的图像帧
  dataset_videos/    数据集抽帧或导出视频
  camera_calibration/相机角色标定截图
  server_result/     推理/服务侧实验输出
```

训练脚本和 agent_factory 默认 checkpoint 根目录应使用 `outputs/checkpoints`。
推理服务器默认模型路径也放在 `outputs/checkpoints/...`。

## 不再新增的位置

不要再把新数据写到这些旧路径：

```text
agent_infra/*/Record/data/
data/online_rl/
demos_RL/
run_results/
critic_only_training/
server_result/
```

如果历史脚本或文档仍引用这些路径，优先改成 `datasets/` 或 `outputs/`。
已有历史文件不需要在代码改动时强行搬家，避免误删或破坏可复现实验。
