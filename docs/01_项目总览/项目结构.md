# 项目结构

本文档用于快速理解 `Arm_studio` 的目录分工。当前项目已经把真实机械臂、仿真、数据、训练和推理放在同一个主仓库中，后续开发应优先保持这些边界清楚。

当前 `Arm_studio` 是项目根目录。外层 `Piper_test/` 已完成代码融合、由用户外部备份并删除，后续不再作为运行依赖；外层 `3DGS/` 暂不移动，等进入 RoboSimGS/3DGS 阶段再纳入根目录规划。

## 核心运行层

```text
agent_infra/
  Piper_Env/        Piper 单臂/双臂真实环境、统一 real/sim 入口、录制与回放
  Realman_Env/      历史 Realman 双臂环境参考

robot/
  base_arm.py
  piper/
    agent.py
    pose_ik.py
    piper_assets/

environments/
  conveyor_env.py
  grasping_env.py

teleop/
  get_pose.py
  real_to_sim.py
```

其中 `agent_infra/Piper_Env` 是后续真实/仿真统一的第一入口；`robot/`、`environments/`、`teleop/` 是 Arm_studio 原有仿真和遥操能力。

## 外部 SDK 与参考代码

```text
pyAgxArm/
  Piper 硬件 SDK，本项目优先使用这一份本地源码

third_party/
  pyorbbecsdk/      Orbbec Python SDK 源码快照，后续 depth SDK 验证时使用
```

默认相机链路不要求安装 `pyorbbecsdk`，Orbbec RGB 优先走 V4L2 fallback。需要 depth 时再从 `third_party/pyorbbecsdk` 安装或构建。

## 产物目录

```text
outputs/

  camera_calibration/
  server_result/

  piper/
  realman/
```

新的数据集和训练缓存应写入 `datasets/`；checkpoint、推理输出、debug 图、视频和评估可视化应写入 `outputs/`，不要散落在仓库根目录。更细规则见 `docs/01_项目总览/目录约定.md`。
