# 训练与数据使用

本文档记录当前 Piper pose-action 数据集和视觉 diffusion policy 的推荐训练流程。

## 1. 当前策略契约

面向数据集保存的动作是 7 维 pose action：

```text
[x, y, z, roll, pitch, yaw, gripper]
```

面向训练的连续动作是 9 维：

```text
[x, y, z, rotation_6d]
```

夹爪不作为连续回归目标训练，而是单独训练为二分类标签：

```text
0 = open
1 = close
```

因此 checkpoint 中通常保存：

```text
model_state
ema_model_state
optimizer
scheduler
scaler
stats
step
best_val_loss
```

推理时，模型会把 9 维连续动作解码回 6 维 pose，再对夹爪输出做阈值判断，最后执行：

```text
pose -> BoundedPiperIK -> pd_joint_pos
```

这样可以避开 ManiSkill 内置 `pd_ee_pose` IK 分支在 Piper 上不稳定的问题。

## 2. 重要配置

主要配置文件：

```text
training/Diffusion_Training/training_config.py
```

当前重要默认值：

```python
image_size = 128
vision_feature_dim = 128
continuous_action_dim = 9
action_representation = "xyz_rot6d_gripper_binary"
prediction_horizon = 32
n_obs_steps = 3
inference_stride = 8
batch_size = 64
save_every = 5000
log_every = 10
num_workers = 2
pretrained_vision = False
```

`pretrained_vision=False` 表示 ResNet18 视觉编码器不会从网络下载权重。如果要使用本地 ImageNet 权重，设置：

```python
vision_weights_path = "/path/to/resnet18-f37072fd.pth"
```

## 3. 准备统一 pose 数据集

当数据目录里混有旧 joint 记录和新 pose 记录时，先统一转换为 pose 数据：

```bash
python scripts/data_tools/build_unified_pose_dataset.py \
  "datasets/*.hdf5" \
  "datasets/auto_collected_real/*.hdf5" \
  --output-dir datasets/unified_pose_all \
  --prefix piper_pose_unified
```

输出文件示例：

```text
datasets/unified_pose_all/piper_pose_unified_0000.hdf5
datasets/unified_pose_all/piper_pose_unified_0001.hdf5
```

每个转换后的文件包含：

```text
action
action_continuous
gripper_label
observation/state
observation/sensor_data/front_view/rgb
observation/sensor_data/hand_camera/rgb
```

转换清单会写入：

```text
datasets/unified_pose_all/manifest.csv
```

## 4. 生成图像缓存

HDF5 懒加载可以避免内存爆炸，但训练时每个 batch 临时 resize 仍然会慢。正式训练前建议先生成 128 图像缓存：

```bash
MPLCONFIGDIR=/tmp/matplotlib \
python scripts/data_tools/build_training_image_cache.py \
  datasets/unified_pose_all \
  --output-dir datasets/unified_pose_all_128 \
  --image-size 128 \
  --compression lzf
```

压缩方式建议：

```text
lzf    默认推荐，随机读取比 gzip 快
none   读取最快，但文件更大
gzip   文件更小，但随机读取更慢
```

训练时推荐读取：

```text
datasets/unified_pose_all_128
```

不要同时把原始目录和缓存目录都传给训练脚本，否则同一批示教会被重复训练。

## 5. 验证数据

检查 pose、FK、IK 是否一致：

```bash
python scripts/data_tools/validate_pose_dataset.py \
  "datasets/unified_pose_all/*.hdf5"
```

如果 IK solver 达到最大迭代次数，但最终位置和旋转误差已经在阈值内，会被视为可接受。

## 6. 导出相机视频

快速检查前视角和手眼相机是否录制正确：

```bash
python scripts/data_tools/export_hdf5_videos.py \
  "datasets/unified_pose_all/*.hdf5" \
  --output-dir outputs/dataset_videos/pose_camera_samples \
  --fps 30 \
  --max-frames 300
```

导出视频拼接格式：

```text
front_view | hand_camera
```

说明：`front_view/hand_camera` 是 ArmStudio ManiSkill 原始相机名；统一 Piper 环境会把它们映射为实机风格的 `base_camera/wrist_camera`，双臂仿真会进一步映射为 `base_camera/left_hand_camera/right_hand_camera`。当前 `hand_camera` 挂载在 URDF 的 `hand_cam` 坐标系上，right hand 使用 `piper_description_with_camera_right.urdf`，left hand 使用 `piper_description_with_camera_left.urdf`，相机外参写在各自 `hand_camera_joint` 中。

## 7. 开始训练

训练前先检查 CUDA：

```bash
nvidia-smi
```

再检查 PyTorch 是否能看到 GPU：

```bash
python - <<'PY'
import torch
print(torch.cuda.is_available())
print(torch.cuda.device_count())
if torch.cuda.is_available():
    print(torch.cuda.get_device_name(0))
PY
```

启动训练：

```bash
MPLCONFIGDIR=/tmp/matplotlib \
python training/Diffusion_Training/train_diffusion_vision.py \
  --data-dir datasets/unified_pose_all_128 \
  --total-steps 100000 \
  --save-every 5000 \
  --num-workers 2
```

常用覆盖参数：

```bash
--batch-size 32
--batch-size 64
--num-workers 0
--num-workers 2
--num-workers 4
```

如果 GPU 利用率低，可以谨慎提高 `num_workers`；如果内存不稳定，就降低它。

## 8. 恢复训练

```bash
MPLCONFIGDIR=/tmp/matplotlib \
python training/Diffusion_Training/train_diffusion_vision.py \
  --data-dir datasets/unified_pose_all_128 \
  --resume outputs/checkpoints/vision/checkpoint_5000.pth \
  --total-steps 100000 \
  --save-every 5000 \
  --num-workers 2
```

checkpoint 和 loss 曲线默认写入：

```text
outputs/checkpoints/vision/
```
