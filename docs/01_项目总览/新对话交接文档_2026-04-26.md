用户已把四个 Piper 机械臂和三个 Orbbec 相机接到电脑。

用户手动激活 CAN 后，接口映射为：

```text
Interface [can_mr] is physically connected to USB device: 1-2.2:1.0
Interface [can_ml] is physically connected to USB device: 1-3.2:1.0
Interface [can_sr] is physically connected to USB device: 1-2.3:1.0
Interface [can_sl] is physically connected to USB device: 1-3.3:1.0
```

语义约定：

- `can_mr`：右主臂 master right
- `can_sr`：右从臂 slave right
- `can_ml`：左主臂 master left
- `can_sl`：左从臂 slave left

`ip -details -statistics link show` 已确认：

- 四个 CAN 都是 `UP`。
- 四个 CAN 都是 `ERROR-ACTIVE`。
- 波特率都是 `1000000`。
- 未发现 bus-off。

2026-04-27 用户确认 `can_sr` 不动的根因是右从臂曾被误设为主导臂。`scripts/piper/can/can_quick_activate.sh` 已在四路 CAN 激活后自动对 `can_sl`、`can_sr` 调用 `set_follower_mode()`，后续重新激活 CAN 时会同步恢复左右从臂模式。

## 8. 当前不能贸然执行的操作

为了硬件安全，新对话不要直接运行以下动作，除非用户明确确认机械臂处于安全姿态、急停可用、人员远离工作空间：

- `env.reset()`
- `env.step(...)`
- 轨迹回放
- 数据采集脚本的真实运动部分
- `move_j`
- `move_p`
- `set_leader_mode`
- 单独对未知接口执行 `set_follower_mode`；`can_quick_activate.sh` 中对 `can_sl/can_sr` 的从臂模式恢复是当前约定入口
- `enable_arm`
- 任何可能导致从臂跟随主臂或机械臂移动的脚本

`PiperEnv` 初始化本身也可能设置模式、速度、加速度等参数，所以不要把“创建真实 env”当作完全只读操作。

当前 `can_quick_activate.sh` 会写入 `can_sl/can_sr` 的从臂模式；其他硬件动作仍按上面的安全约束处理。
