# 代码掌控地图

这份文档的目的不是替代源码，而是帮你建立“我知道从哪里进、数据怎么走、改哪里会影响谁”的开发掌控感。每读懂一个模块，就可以继续补充本文件。

## 0. 总体主线

当前工程最重要的主线是：

```text
config
  -> environment factory
  -> real/sim env
  -> obs/action/meta_keys
  -> recorder/postprocess
  -> dataset loader
  -> policy train/inference
  -> real/sim execution
```

你每次读代码时，都可以问三个问题：

```text
入口在哪里？
输入输出是什么？
谁依赖这个输出？
```

## 1. 统一环境入口

核心文件：

```text
agent_infra/Piper_Env/Env/unified_piper_env.py
agent_infra/Piper_Env/Env/protocol.py
agent_factory/env/env_factories.py
agent_infra/Piper_Env/Config/unified_real_single.yaml

agent_infra/Piper_Env/Config/unified_sim_single.yaml
agent_infra/Piper_Env/Config/unified_sim_dual.yaml
```

你必须掌握：

```text
make_unified_piper_env() 如何读取 config
backend=real|sim 如何分流
arm_mode=single|dual 如何分流
control_mode=joint|pose 如何进入 action 语义
meta_keys 如何保证真实和仿真字段一致
```

改动边界：

```text
新增真实/仿真 backend 时，优先从 make_unified_piper_env() 接入。
新增 obs/action 字段时，必须同步 protocol、recorder、postprocess、训练 loader。
```

掌握验收：

```text
你能说明 unified_sim_dual.yaml 最终创建了哪个 Env 类。
你能说明一个 action 从外部 step() 进入后如何被拆成左右臂命令。
```

## 2. 真实 Piper 环境层

核心文件：

```text
agent_infra/Piper_Env/Env/utils/piper_base_env.py
agent_infra/Piper_Env/Env/piper_camera_wrapper.py
agent_infra/Piper_Env/Env/single_piper_env.py
agent_infra/Piper_Env/Env/dual_piper_env.py
agent_infra/Piper_Env/Camera/
agent_infra/Piper_Env/Config/single_piper_config.yaml
agent_infra/Piper_Env/Config/dual_piper_config.yaml
```

你必须掌握：

```text
PiperBaseEnv 负责机械臂连接、状态读取、动作下发。
PiperEnv 增加主臂遥操、人机接管、actual_action。
PiperCameraWrapper 增加相机 obs。
SinglePiperEnv / DualPiperEnv 是真实环境最终入口。
```

改动边界：

```text
不要让相机逻辑进入机械臂 base env。
不要让训练代码直接依赖具体相机 SDK。
真实控制 safety guard 应优先写在 Env 层，而不是训练层。
```

掌握验收：

```text
你能说明 master arm 和 follower arm 分别从 config 的哪个字段读取 CAN 口。
你能说明 info["actual_action"] 为什么对遥操采集重要。
```
相机视角对应的编号
```text
CH27152011E -> base_camera / 第三视角
CH28822007C -> left_hand_camera / 左腕视角
CH2592100GJ -> right_hand_camera / 右腕视角
```

仿真腕部相机对应关系：

```text
piper_arm_left  -> piper_description_with_camera_left.urdf  -> left_hand_camera
piper_arm_right -> piper_description_with_camera_right.urdf -> right_hand_camera
```


## 3. 仿真环境层

核心文件：

```text
agent_infra/Piper_Env/Env/unified_piper_env.py
environments/
robot/piper/agent.py
robot/piper/pose_ik.py
scripts/diagnostics/viewenv.py
```

当前状态：

```text
SinglePiperSimEnv 已作为 Arm_studio 单臂 ManiSkill 环境适配器。
DualPiperSimEnv 目前仍是协议层双臂，由两个单臂场景组合。
真正共享物理世界的双臂 ManiSkill/SAPIEN 场景仍未完成。
```

你必须掌握：

```text
仿真 obs 如何被转成和真实 obs 一致的 key。
仿真 action 如何从统一协议转成 ManiSkill controller action。
相机 RGB 如何进入 obs/rgb/...。
```

改动边界：

```text
共享双臂场景应该替换 DualPiperSimEnv 内部实现，但保持外部 obs/action 协议不变。
不要让训练 loader 关心仿真内部 controller 名称。
```

掌握验收：

```text
你能说明当前 dual sim 为什么还不是真正双臂物理交互。
你能列出实现共享场景双臂仿真要新增哪些资产和 scene 逻辑。
```

## 4. 数据协议、录制与转换

核心文件：

```text
agent_infra/Piper_Env/Record/recorder.py
agent_infra/Piper_Env/Record/replay.py
agent_infra/Piper_Env/Record/postprocess.py
data/recorder.py
scripts/data_tools/build_unified_pose_dataset.py
scripts/data_tools/build_training_image_cache.py
scripts/data_tools/validate_pose_dataset.py
scripts/data_tools/export_hdf5_videos.py
```

你必须掌握：

```text
raw H5 推荐结构：obs/... + action/... + meta/env_meta。
merged H5 如何组织 traj_0、traj_1。
H5 和 LeRobot 如何互转。
旧 Arm_studio 数据目前为何还需要转换器。
```

改动边界：

```text
新增字段必须写入 env_meta，不能只写 dataset。
真实数据和仿真数据必须保持同一字段语义。
训练缓存只能加速读取，不应改变源数据语义。
```

掌握验收：

```text
你能打开一个 H5 后说清 obs、action、meta 各自作用。
你能说明 replay 依赖哪些 action key。
```

## 5. 训练与模型层

核心文件：

```text
training/Diffusion_Training/training_config.py

models/DiffusionPolicy/policy.py

```

你必须掌握：

```text
训练数据中的 7D pose action 如何转成 xyz + rot6d + gripper_label。
vision encoder 输出如何和状态/action history 拼接。
checkpoint 中保存哪些状态。
推理时如何从模型输出回到真实/仿真可执行 action。
```

改动边界：

```text
先让 diffusion 训练稳定读取统一数据，再合并更复杂的 agent_factory 训练。
不要同时改模型结构、数据格式和环境接口，否则问题难定位。
```

掌握验收：

```text
你能说明一个 batch 从 H5 到 model.forward() 的 shape 变化。
你能说明 gripper 为什么不用连续回归作为唯一训练目标。
```

核心文件：


说明：历史 `scripts/run_inference.py` 和断掉的 `scripts/inference.sh` 已删除；local/remote 闭环推理需要后续恢复 runner 或接入新的执行器。

你必须掌握：

```text

runner 如何把策略接到控制循环和 H5 保存。
HITL 人机接管如何影响实际写入的数据。
```

改动边界：

```text
实时控制循环中不要加入耗时阻塞操作。
远程推理协议必须明确输入输出 shape。
```

掌握验收：

```text
你能说明 inference/client.py 发送了什么，接收了什么。
你能说明 target_fps、inference_stride、prediction_horizon 的区别。
```

## 7. 3DGS 与 SAPIEN/ManiSkill 增强层

参考目录：

```text
../3DGS/README.md
../3DGS/Gaussians/render.py
../3DGS/utils/icp.py
../3DGS/Articulation/
```

未来建议新增：

```text
assets/realsim/
sim_assets/
gs_bridge/
```

你必须掌握：

```text
3DGS 只负责视觉背景或视觉增强，不负责物理。
物体交互必须使用 mesh/URDF 和 SAPIEN/ManiSkill 碰撞。
ICP/world_alignment 负责把真实重建和仿真世界对齐。
Camera pose alignment 不能依赖 RoboSimGS 未公开代码，应先做人工/半自动标定。
```

改动边界：

```text
不要直接把 3DGS/DataGeneration 搬进来，它绑定 Genesis 和固定路径。
先做离线资产规范和渲染桥，再接入在线采集。
```

掌握验收：

```text
你能说明 Gaussian PLY、mesh.obj、URDF、physics.json、cameras.json、world_alignment.json 各自用途。
你能说明为什么 GS 背景不能替代碰撞几何。
```

## 9. 每次改代码前的五行契约

建议你每次动手前写下：

```text
我要改的入口：
我要保持不变的数据格式：
我要新增/删除的字段：
可能影响的脚本：
验证命令：
```

这五行会把“凭感觉改代码”变成“有边界地改代码”。掌控感就是这样一点点长出来的。
